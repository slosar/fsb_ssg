# Science validation

This directory holds the **simulation-based validation** of `fsb_ssg` and the
report that documents it. Unlike the unit tests (which run on synthetic maps in
seconds), these are exhaustive science checks: full-sky ground truth, a
400-realization masked + noisy run, the analytic transfer function by four
independent methods, a NaMaster Gaussian-covariance cross-check, and the figures
for the report.

The simulation maps are **not** included in the repository (they are tens of GB);
point `FSB_SIM_DIR` at a directory that contains them.

```
pipeline/   staged measurement scripts + the simlib adapter
report/     the LaTeX report and its figures
```

## Requirements

* `fsb_ssg` and its dependencies installed (`pip install -e ..` from the repo
  root, plus the `science` extras: `pip install -e "..[science]"` for matplotlib).
* The simulation maps in `FSB_SIM_DIR`, named as the adapter expects:
  * `map_g1_z1.0334_r{NNN}_nside1024.fits.gz`, `map_g2_z1.0334_...` — source shear;
  * `map_delta_z0.5406_r{NNN}_nside1024.fits.gz` — lens density (z≈0.54);
  * `map_delta_source_z1.0_r{NNN}_nside1024.fits.gz` — source density (noise model);
  * `desy6_map{1..4}_ns1024.fits.gz` — the four disjoint DES Y6 footprints.

The maps, redshift tags and noise parameters can be changed in
[`pipeline/simlib.py`](pipeline/simlib.py).

## Running the pipeline

```bash
cd pipeline
export FSB_SIM_DIR=/path/to/simulation/maps
export FSB_PYTHON=python          # interpreter with fsb_ssg + pymaster

FSB_NSIDE=256  bash run_pipeline.sh    # fast validation (one filter band)
FSB_NSIDE=1024 bash run_pipeline.sh    # production (three bands; masks in parallel)
```

Filter bands are selected automatically: any default band with upper edge below
`lmax = 2·nside`. Intermediate products are written to `pipeline/data/`
(git-ignored) and figures to `report/figures/`.

### Stages

| stage                | what it does |
|----------------------|--------------|
| `build_cache.py`     | degrade & cache the input maps to the working NSIDE (skipped when native) |
| `ground_truth.py`    | full-sky, noise-free, unit-weight ground-truth bispectrum |
| `run_masked.py`      | 100 sims × 4 disjoint masks = 400 noisy realizations, three schemes |
| `combine_masks.py`   | merge the per-mask outputs |
| `analyze.py`         | unbiasedness χ², measured transfer `T_A`, SNR, correlation matrices, figures |
| `predict_transfer.py`| analytic transfer (B) Gaussian-MC and (C) closed form + fiducial cross-spectra |
| `method_d.py`        | (D) dummy-window mode-coupling calibration (transfer-free) + figures |
| `nmt_covariance.py`  | NaMaster Gaussian covariance vs the 400-realization sample scatter |

Useful environment variables: `FSB_NREAL` (number of realizations),
`FSB_MASKS` (comma-separated mask subset for parallel runs), `FSB_BANDS`,
`OMP_NUM_THREADS`. A quick smoke run:

```bash
FSB_NSIDE=256 FSB_NREAL=5 FSB_MASKS=0 FSB_BANDS=1 python run_masked.py
```

## Building the report

```bash
cd report
pdflatex bispectrum_test.tex && pdflatex bispectrum_test.tex   # twice for cross-refs
```

`bispectrum_test.tex` is the main validation report; `optimal_weights.tex` is the
companion note deriving the optimal weights `W1`, `W2`. The figures in
`report/figures/` are regenerated by the pipeline stages above.
